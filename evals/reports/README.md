# evals/reports

Generated output from `pnpm evals:run` and `pnpm evals:report`. Three formats, one per file.

## eval-report.json

Full structured report. Contains the `EvalReport` object: summary stats, per-pipeline / per-suite / per-scorer aggregations, common failure reasons, worst cases, and the complete `executions[]` array with per-case scorer results and traces.

The trace for each execution includes `systemPrompt` and `rawResponse` fields when the executor captures them. These are always present in modular pipeline runs and contain the literal text sent to and received from the model.

## eval-report.md

Markdown tables for quick scanning in a terminal, PR summary, or diff review. Generated by `renderMarkdownReport` from the JSON.

## eval-report.html

Self-contained HTML file. Sticky header with headline stats, aggregation cards with inline bar charts, operational metrics, filterable/sortable case list. Each case expands to show model, tokens, latency, scorer table with weights and diagnostics, validation stages, retrieval hits, system prompt (collapsible), assembled context (collapsible), and raw model response (collapsible). Vanilla JS for filter and sort, no external assets.

Open it directly in a browser after running evals:

```bash
pnpm evals:run
open evals/reports/eval-report.html
```
